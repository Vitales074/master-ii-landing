🌐
Для урока 4 — лендинг с нуля

Когда сайт уже сгенерирован по вашему первому промпту и вы хотите довести дизайн до референса или проверить результат — эти два промпта экономят время на итерациях.

Реализовать дизайн по картинке и проверить самому implement this design, then take a screenshot of the result, compare it to the original, and fix any differences
Когда использовать: вставьте картинку-референс в чат вместе с этим промптом. Claude сам сверит результат со скриншотом и исправит расхождения — не нужно самому тыкать пальцем «здесь не так».

🏗
Для урока 5 — многостраничный сайт

Прежде чем создавать структуру из нескольких страниц, полезно сначала увидеть план — без этого Claude Code иногда начинает строить не в ту сторону.

Спланировать структуру, не трогая код plan how to build a multi-page site for [тема]: list the pages and structure, but don't edit anything yet Пример: ...for a barbershop site: list the pages and structure, but don't edit anything yet
Точечно исправить визуальный баг the [элемент] extends beyond the [контейнер] on mobile. fix it.
Второй промпт — готовый шаблон для любой мелкой поломки вёрстки на телефоне. Подставьте название элемента вместо квадратных скобок.

🤖
Для урока 6 — агенты и контент

Это прямое попадание в идею урока: вместо длинного промпта пусть Claude сам расспросит вас, а досье бренда собирается по образцу ваших лучших текстов.

Собрать техзадание через интервью I want to build [агент для контента]. interview me about implementation, UX, edge cases, and tradeoffs until we have covered everything, then write the spec to SPEC.md
Научить агента вашему голосу по примерам read the [примеры постов] in [папка] to learn the structure and voice, then draft a new one for [тема] Пример: read the last 10 posts in /content/instagram to learn the structure and voice, then draft a new one about a spring discount

⚡
Для урока 8 — первое приложение

Тот же принцип «внутренний инструмент без фреймворков», что и в демо урока — годится для любой идеи, которая приходит в голову. Второй промпт показывает, как Claude сам находит и чинит баги.

Создать инструмент с нуля create a [tool] using HTML, CSS, and vanilla JavaScript, then open it in my browser
Написать тесты и сразу исправить всё, что не работает write tests for [фича], run them, and fix any failures
Совет: если хотите увидеть, как Claude сам тестирует и правит баги (как в демо урока 8) — этот второй промпт даёт ровно такое поведение.
Откуда это взято: все промпты — из официальной библиотеки Anthropic для Claude Code (code.claude.com/docs/en/prompt-library), сгруппированной по этапам разработки. Полный список — на сайте, там же можно фильтровать по тегам (Build, Debug, Review и другие).

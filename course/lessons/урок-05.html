Многостраничный сайт:
профессиональный подход

Лендинг освоен — переходим на следующий уровень. Сайт из нескольких страниц, документация проекта, работа с большими объёмами контента. Здесь же — технология reverse-инжиниринга: просим Claude самому предложить архитектуру.

⏱ ~60 минут
💻 Claude Code
🏗 5–10 страниц

📋
Чек-лист перед стартом

Прежде чем дать первую команду, убедитесь, что всё готово. Если сделать что-то не по порядку — придётся переделывать.

  • VPN включён (AmneziaVPN), IP — Нидерланды
  • Создана новая папка под проект (не та же, что лендинг)
  • Папка открыта в Claude Code, нажато «Trust»
  • Выбрана модель Sonnet, Effort High
Правило: новый проект — новая папка. Не смешивайте лендинг и многостраничный сайт в одной директории. Claude может запутаться в файлах, а вы потеряете время на разбор конфликтов.

🔄
Метод reverse-инжиниринга

Большинство людей объясняют Claude, что нужно делать. Более эффективный подход — попросить Claude самому предложить решение, а потом подкорректировать его план.

Как обычноМетод reverse-инжиниринга
«Сделай главную страницу, страницу услуг, страницу контактов и портфолио» «Я хочу сайт для частного архитектора. Предложи оптимальную структуру страниц — объясни, зачем каждая»
«Добавь форму обратной связи» «Как лучше всего организовать получение заявок на этом сайте? Предложи варианты»
«Сделай анимацию» «Какие анимации улучшат восприятие этого сайта, не перегружая его?»
Почему это работает лучше: Claude предлагает структуру исходя из логики пользователя, а не шаблона. Его план часто включает детали, о которых вы не подумали — страницу FAQ, страницу проектов с фильтрацией, блог.

💬
Промпт из 6 компонентов

Для многостраничного сайта промпт строится по той же логике, что для лендинга, но добавляется несколько важных деталей.

6 компонентов грамотного промпта Роль: Ты опытный веб-разработчик и UX-дизайнер
Контекст: [Кратко о проекте, для кого сайт, цель]
Задача: Создай многостраничный сайт — главная, услуги, портфолио, контакты
Формат: Отдельный HTML-файл на каждую страницу, единый style.css
Ограничения: Без внешних зависимостей, адаптив, кроссбраузерность
Пример: [Ссылка или описание референсного сайта — опционально]
Почему лучше набирать руками: процесс набора помогает уточнить детали. Часто по ходу написания добавляешь важные подробности, которые пропустил бы при копировании.

📄
Файл документации: замена памяти

Чем больше проект — тем важнее иметь файл с описанием. Когда контекст Claude обнуляется (через несколько часов или при переоткрытии Code), этот файл позволяет восстановить всю картину за секунды.

01
Создайте файл readme.md в папке проекта
02
Попросите Claude заполнить его

«Опиши наш проект в этом файле: структура страниц, цветовая палитра, шрифты, логика навигации, что ещё не сделано». Claude сам напишет подробный отчёт по тому, что уже создано.

03
В новой сессии — первый делом откройте этот файл

В Claude Code напечатайте @readme.md — Claude прочитает файл и сразу войдёт в контекст проекта. Не нужно объяснять всё заново.

Как примерно выглядит заполненный readme.md:

Пример readme.md (сокращённо) # Сайт архитектора Иванова Страницы: index (главная), services (услуги), portfolio (12 проектов с фильтром), contacts (форма + карта) Дизайн: тёмно-серый фон #1a1a1a, акцент — терракотовый #c96f4a, шрифт Manrope Сделано: все 4 страницы, навигация, адаптив Осталось: заменить фото-заглушки в портфолио, подключить форму к почте
Обновляйте файл в конце каждой сессии: «Обнови readme.md — добавь что мы сделали сегодня и что осталось». Это займёт 20 секунд и сэкономит час в следующий раз.

⚙️
Режимы работы: когда что включать

РежимКогда использоватьЧто происходит
Ask Permission (по умолчанию) Небольшие правки, ранние этапы Claude спрашивает перед каждым действием
Plan Mode Перед большой задачей — новая страница, рефакторинг Сначала составляет план, вы одобряете
Bypass / Auto Mode Когда задача чёткая и срочная, доверяете Claude Делает всё без остановок на подтверждения
Про токены: многостраничный сайт потребляет много контекста — примерно 60% лимита за один крупный проект. Если лимит заканчивается в середине работы — не паникуйте. Сохраните прогресс через readme.md и продолжите в новой сессии.

🚀
Публикация: Vercel

Для многостраничных сайтов удобнее Vercel — он лучше обрабатывает навигацию между страницами. Netlify тоже работает, но Vercel настраивается за 4 шага.

01
Регистрация на vercel.com через GitHub или Google
Форма регистрации Vercel — проще всего нажать «Continue with Google»
02
New Project → Import → выбираем папку
03
Framework preset: Other (не Next.js!)

Выберите «Other» — иначе Vercel попробует собрать проект как React-приложение.

04
Deploy → получаем ссылку *.vercel.app

Частые вопросы

Claude отвечает на русском языке, а мне нужно всё на английском
Просто начните писать промпты на английском — Claude автоматически переключится на английский для кода и комментариев. Или добавьте в промпт: «All code comments and content in English».
Можно ли передать в Claude файл с данными заказчика (прайс-лист в Excel)?
Да — Claude Code читает Excel, CSV, JSON, Word и другие текстовые форматы. Видео и исполняемые файлы не поддерживаются. Положите файл в папку проекта и напишите: «Используй данные из price.xlsx для таблицы услуг».
Говорить ли клиенту, что сайт сделан с помощью ИИ?
Это ваше решение. Клиента интересует результат — работающий сайт за разумные деньги в срок. Большинство инструментов в нашем деле — это «помощники», которые мы не перечисляем. С другой стороны, прозрачность строит доверие. Если спросят — не скрывайте.

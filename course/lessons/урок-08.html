Если подписка перестала работать

Иногда Claude Code показывает красную плашку «subscription is paused» или «payment failed». Это не поломка — незакрытый счёт.

01
Нажмите на плашку → Manage subscription
02
Pay → подтвердите платёж
03
Перезайдите в аккаунт

Выйдите и зайдите снова — статус обновится.

Совет: держите подписку на карте, которую не используете для других подписок — так она не попадёт в случайную блокировку.

Стартовый чек-лист: 6 шагов

Прежде чем дать первую команду — убедитесь, что всё готово. Этот порядок нельзя нарушать: одна пропущенная строчка — и час переделок.

  • Открыт Claude Code (не Chat)
  • Создана и открыта отдельная папка для проекта
  • Папка добавлена в Code, нажато «Trust»
  • Выбрана модель Sonnet, Effort High
  • Режим работы: Plan Mode для новых задач
  • Задача сформулирована чётко, перед запуском

Plan Mode vs Auto: когда включать голову заранее

Аналогия со строительством: можно начать класть кирпичи без чертежа — а потом сносить, когда окажется, что стена не там. А можно потратить час на проект — и строить правильно с первого раза.

Auto ModePlan Mode
Когда использовать Небольшие правки, понятная задача Новое приложение, сложная логика
Что происходит Сразу начинает делать Сначала составляет план, вы одобряете
Риск Может пойти не туда Минимальный — вы видите маршрут заранее
Скорость Быстрее на старте Медленнее на старте, быстрее в итоге
Как включить Plan Mode: в строке ввода Claude Code нажмите на иконку с шестерёнкой или переключите режим на «Plan». Claude сначала задаст уточняющие вопросы → составит план → покажет кнопку «Accept and continue».

Демо: трекер задач за 15 минут

Промпт целиком — можете скопировать и повторить демо у себя:

Промпт демо-задачи Создай систему учёта клиентских заявок для небольшой компании. Нужна таблица с заявками, статусы (новая / в работе / завершена), фильтры по статусу, приоритет, возможность экспорта в CSV. Один HTML-файл, данные храни в localStorage, адаптивная вёрстка.

Что Claude сделал в Plan Mode:

  • ▸Задал 3 уточняющих вопроса (нужна ли авторизация? данные хранить локально или в БД? мобильная версия?)
  • ▸Составил план: 5 файлов, логика хранения в localStorage, CSS на Tailwind
  • ▸После одобрения — перешёл в Auto Mode и создал всё без остановок
  • ▸Сам открыл браузер, нажал кнопки, нашёл баг с сортировкой — исправил
Итог: таблица заявок, три статуса с цветовой маркировкой, фильтры, приоритет, экспорт CSV, адаптив — 14 минут 37 секунд. Никакого кода руками.

5 форматов готового приложения

ФорматКак использоватьАудитория
Localhost (локальный сервер) Открыть по адресу http://localhost:3000 — только на вашем компьютере Только вы
HTML-файл Открыть файл index.html в браузере — без сервера, но без динамики Только вы
Netlify / Vercel Загрузить папку на хостинг — получить ссылку Любой по ссылке
Telegram-бот Подключить через API — урок 11 Пользователи Telegram
VPS-сервер Деплой на сервер — работает 24/7 — урок 9 Любой по ссылке, 24/7
Localhost — это НЕ ссылка для клиента. Адрес http://localhost:3000 работает только на вашем компьютере. Чтобы отправить ссылку другому человеку — нужен деплой на хостинг или сервер.

Sonnet vs Opus + Effort: расходуем осознанно

СвязкаДля чегоЛимит
Sonnet + Low/Medium Правки, небольшие задачи, черновик Расходуется медленно
Sonnet + High Новое приложение, сложная логика — рекомендуется Умеренный расход
Opus + High Финальная полировка, сложные баги, критичные задачи Расходуется быстро
Рекомендация: для большинства задач курса — Sonnet + High. Opus держите в запасе для ситуаций, когда Sonnet несколько раз не справился с задачей.

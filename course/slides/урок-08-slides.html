Подписка не работает — что делать

Красная плашка «subscription is paused» или похожее сообщение — это не поломка.

1Нажмите на плашку или перейдите в настройки аккаунта
2Найдите кнопку Pay или Update payment
3Подтвердите оплату — карта уже привязана
4Перезайдите в аккаунт — всё восстановилось
Совет: заведите отдельную карту под подписку Claude — так уведомление о незакрытом платеже придёт точно вам.

Стартовый чек-лист: 6 шагов

1Code открыт, не Chat
2Папка проекта создана и открыта
3Trust нажат — разрешение на работу с папкой
4Модель выбрана (Sonnet — стандарт)
5Режим настроен (Plan Mode для сложных задач)
6Задача сформулирована — что именно создаём
Этот порядок не менять. Правильный старт сэкономит час правок в конце.

Auto Mode vs Plan Mode

⚡ Auto Mode

Выполняет сразу. Хорошо для простых задач — добавить кнопку, изменить цвет, написать текст. Быстро, но без предсказуемости.

📋 Plan Mode

Сначала составляет план: задаёт вопросы → пишет список шагов → ждёт одобрения → выполняет. Для приложений с нуля обязательно.

Аналогия: Plan Mode — архитектор рисует чертёж перед строительством. Без плана можно построить, но переделывать дорого.

Как Claude тестирует приложение

После написания кода Claude не останавливается — он сам его проверяет:

→Запускает приложение в браузере
→Нажимает кнопки, заполняет формы
→Смотрит консоль на ошибки
→Находит проблему → сам исправляет → перепроверяет
Это не магия — Claude использует инструменты браузера точно так же, как делал бы разработчик. Вы видите процесс в реальном времени.

5 форматов вывода готового приложения

LocalhostРаботает на вашем компьютере. НЕ ссылка для других — только для вас.
HTML-файлСтатичное приложение — открывается в браузере. Нет сервера, подходит для простых инструментов.
Онлайн-серверNetlify/Vercel — публичная ссылка. Для клиентов и портфолио.
Telegram-ботРаботает в мессенджере. Урок 11 — подробнее.
.exe файлУпаковать как установщик для Windows/Mac через Electron или PyInstaller.

Модели и Effort: расходуем токены осознанно

Sonnet

Стандарт для большинства задач. Баланс скорость/качество. Экономит лимиты. Сайты, приложения, тексты.

Opus

Максимальная мощность. Тратит больше токенов. Для сложной архитектуры, финальной полировки, нестандартных задач.

Effort: Low–Medium = обычные задачи. High = крупный проект. Smarter = самая медленная, самая умная. Выбирайте под задачу, не «всегда максимум».

Ключевые выводы

  • Claude сам тестирует: запускает, нажимает, исправляет
  • 5 форматов вывода: localhost ≠ публичная ссылка

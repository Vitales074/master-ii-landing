Telegram vs веб-сайт: зачем оба

🌐 Веб-приложение

Нужно открыть браузер и перейти по ссылке. Нет push-уведомлений. Широкие возможности интерфейса.

✈️ Telegram-бот

Уже установлен у большинства. Push-уведомления. Ответ в чате. Нет регистрации для пользователя.

Один API-ключ работает и там, и там — это просто разные «трубы» к одной модели. Не нужно создавать отдельного агента для каждого канала.

memory.md: контекст между сессиями

Проблема: Claude Code не помнит прошлые сессии. Решение — файл memory.md.

Команда в конце сессии: «Запомни контекст, продолжу завтра» → Claude создаёт memory.md: что сделано, архитектура, что осталось.
1В начале новой сессии: «@memory.md — прочитай контекст»
2Claude читает файл и продолжает с того места где остановился
3Обновляйте файл в конце каждой сессии
memory.md работает не только для ботов — для любого проекта в Code.

BotFather: регистрация бота

BotFather — официальный бот Telegram для создания ботов. Найдите @BotFather в поиске.

1Откройте Telegram → найдите @BotFather
2Отправьте команду /newbot
3Введите отображаемое имя бота (произвольное)
4Введите username: только латиница, обязательно заканчивается на bot
5BotFather выдаёт токен — скопируйте сразу
Важно: кириллица в командах не работает. Имя может быть любым, username — только латиница.

Что Claude строит автоматически

После передачи задачи и токена Claude создаёт:

  • Папку tg-bot/ с нужной структурой файлов
  • Устанавливает библиотеки (python-telegram-bot или node-telegram-bot)
  • Обработчики команд: /start, /help и кастомные
  • Кнопки-меню с нужными разделами
  • Подключение к Anthropic API
  • Тестирует — пишет сам себе в бот
Пример: агент поддержки магазина — кнопки «Размеры», «Доставка», «Возврат», «Статус заказа» + свободный чат.

Типичные ошибки — Claude решает сам

История стирается между сессиями→ Claude добавляет SQLite для хранения
Долгий ответ без обратной связи→ Добавляет прогресс-индикатор «Думаю…»
Flood Control от Telegram→ Убирает конкурирующую фоновую задачу
Кириллица в командах не работает→ Переименовывает команды на латиницу
Принцип: Claude сам находит ошибку, предлагает fix, применяет. Вы только одобряете.

Файл .env: сейф для ключей

В файл .env кладутся все секреты проекта — токены, ключи, пароли.

!Никогда не вставлять ключи прямо в код
!Добавить .env в .gitignore — иначе попадёт в репозиторий
!Отзывать после демонстраций: /revoke в чате с BotFather
!Аудит перед выходом в продакшен
Отличие от SSH-ключа: токен — строка для API-вызовов. SSH-ключ — криптографический файл для подключения к серверу. Разные инструменты для разных задач.

Ключевые выводы

Часть первая

Подготовка рабочей среды

Чек-лист перед стартом любого серьёзного проекта

Перед каждым проектом: VPN включён, папка создана, открыта в Code и нажат Trust, модель и режим настроены. Звучит просто, но один пропущенный шаг может привести к путанице или ошибке. Папка нужна только когда создаёте файлы — для обсуждения идей папка не обязательна.

Правило: 1 проект = 1 папкаНе смешивать разные проекты — потеряется контекст
Большинство начинающих пишут длинный список что нужно сделать. Это работает, но медленно. Reverse-подход другой: сначала спрашиваешь Claude что он предложит. Например: «Мне нужен сайт для фотографа — что предлагаешь включить?» Claude сам предложит структуру страниц, функции, навигацию. Вы оцениваете — оставляете что нужно, убираете лишнее. Это быстрее, чем диктовать каждый шаг.

02
Часть вторая

Reverse-инжиниринг

Пусть Claude думает за вас — вы только одобряете решение

Шесть компонентов. Роль: скажите Claude кем он должен быть. Контекст: для кого сайт. Задача: что конкретно создать. Формат: HTML, без фреймворков. Ограничения: мобильная вёрстка, скорость. Пример: референс стиля. И важное замечание: лучше набирать промпт с клавиатуры, не копировать. Когда печатаете — уточняете мысль, убираете лишнее.

Принцип: не объяснять —
просить предложить

СитуацияЧто делать
Не знаете, что писать в ТЗПопросите Claude проанализировать нишу
Не знаете, какие скиллы нужны«Какие скиллы нужны для 3D-анимации?»
Не нравится дизайнОпишите что не нравится — Claude предложит план
Есть вопрос по настройкеСпросите Claude — точнее, чем Google
Основная проблема Claude: он не помнит прошлые сессии. Решение — файл project.md. Попросите Claude помочь его составить: он задаст вопросы, вы ответите, он сформирует ТЗ. Сохраните как project.md в папку. В следующей сессии напишите «@project.md — читай как контекст» — и Claude сразу в теме. В документ входит: цель, аудитория, список страниц, дизайн-код, технические требования.

Часть третья

Структура промпта

6 компонентов — чем подробнее, тем точнее результат

Лендинг — один HTML-файл, одна цель, один поток. Многостраничный — папка с файлами: index, about, services, contacts. Навигация между страницами. Порядок работы: сначала предложите структуру, одобрите, потом Claude создаёт страницы последовательно. Проверяйте каждую перед переходом к следующей.

Пример

Ссылка-референс или описание стиля

Три режима. Ask Permission — по умолчанию, Claude спрашивает перед каждым действием, хорошо для обучения. Plan Mode — сначала составляет план, вы одобряете, потом выполняет — для крупных проектов. Включается через Ctrl+Shift+P. Bypass — без вопросов, быстро, но рискованно — только если доверяете задаче.

Часть четвёртая

Файл ТЗ и многостраничный сайт

Один утверждённый документ → один промпт → готовый сайт

Vercel — ещё один бесплатный хостинг, отлично подходит для многостраничных сайтов. Регистрируемся через Google. Add New Project, загружаем папку, нажимаем Deploy. Ссылка готова. Vercel и Netlify — оба работают, оба бесплатны. Vercel чуть лучше для многостраничных, Netlify удобнее для одностраничных. Выбирайте любой — разница минимальная.

Публикация на Vercel

vercel.com → Sign Up → «Continue with Google». Дальше New Project → Import → Deploy
Vercel — второй бесплатный хостинг, удобнее для многостраничных сайтов. Вот форма регистрации: проще всего нажать «Continue with Google» — новый пароль не нужен. После входа: Add New Project, импортируете папку, framework preset ставите Other, нажимаете Deploy — и получаете ссылку вида имя.vercel.app.

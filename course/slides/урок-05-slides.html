Чек-лист перед стартом

✓VPN включён — IP из нужной страны
✓Папка проекта создана, открыта в Claude Code, нажат Trust
✓Модель выбрана под задачу (Sonnet для кода, Opus для сложной архитектуры)
✓Режим работы настроен (Plan Mode для крупных задач)
Папка нужна только когда создаёте файлы. Для обсуждения идей можно работать без папки.

Reverse-инжиниринг: пусть Claude предложит план

Вместо того чтобы объяснять как делать — попросите Claude предложить решение самому.

❌ Старый подход

«Создай navbar с 5 пунктами, добавь анимацию, сделай адаптив, потом...»

✓ Reverse-подход

«Мне нужен сайт для фотографа. Что ты предложишь включить? Опиши структуру и обоснуй.»

Claude сам предложит структуру, страницы, функции — вы оцениваете и корректируете. Это быстрее, чем диктовать каждый шаг.

Формула промпта: 6 компонентов

[Роль] Ты senior frontend-разработчик
[Контекст] Сайт для частного тренера по йоге
[Задача] Создай многостраничный сайт с 5 страницами
[Формат] HTML + CSS + JS, без фреймворков
[Ограничения] Мобильная вёрстка, скорость загрузки важна
[Пример] Стиль как у [описание референса]
Почему лучше печатать, не копировать: во время набора вы уточняете мысль, убираете лишнее. Напечатанный промпт обычно точнее скопированного.

Файл документации проекта

Создайте ТЗ-файл один раз — и все сессии будут «помнить» контекст.

1Попросите Claude: «Помоги составить ТЗ для этого проекта в виде .md файла»
2Claude задаёт вопросы → вы отвечаете → он формирует документ
3Сохраните файл как project.md в папке проекта
4В новой сессии: «@project.md — читай это как контекст» → Claude всё знает
Что входит в ТЗ: цель, аудитория, страницы, дизайн-код, технические требования, тексты-заглушки.

Лендинг vs Многостраничный сайт

Лендинг

Один HTML-файл. Одна цель: конверсия. До 10 секций. ~30 мин работы.

Многостраничный

Папка с файлами: index, about, services, portfolio, contacts. Навигация между страницами. ~30–60 мин работы.

Порядок создания: сначала попросите предложить структуру → одобрите → Claude создаёт все страницы последовательно → вы проверяете каждую.

Режимы работы Claude Code

Ask Permission — по умолчанию

Claude спрашивает перед каждым действием. Контроль максимальный. Для обучения и небольших задач.

Plan Mode — для сложных проектов

Claude сначала составляет план, вы одобряете, потом выполняет. Меньше неожиданностей. Включить: ⌘+Shift+P / Ctrl+Shift+P.

Bypass (Auto) — осторожно

Выполняет без вопросов. Быстро, но можно пропустить неправильное решение. Только если доверяете задаче.

Публикация на Vercel: 4 шага

1Зарегистрируйтесь на vercel.com через Google-аккаунт
2На главной нажмите «Add New Project» → выберите «Browse» для загрузки папки
3Загрузите папку проекта → Vercel автоматически находит index.html
4Нажмите Deploy → получите ссылку вида project-name.vercel.app
Vercel и Netlify — оба бесплатны и подходят. Vercel чуть лучше для многостраничных проектов, Netlify — для одностраничных. Выбирайте любой.

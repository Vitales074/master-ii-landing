Три аудитории — три уровня доверия

Близкие

Доверяют без доказательств. Первые клиенты и кейсы.

Знакомые

Нужны примеры работ. Скриншоты, ссылки, описание.

Незнакомые

Нужны кейсы, отзывы и профессиональный сайт.

Как набить портфолио с нуля: проекты для себя, семьи, друзей → реальные скриншоты и ссылки → попросить написать отзыв. 3–5 кейсов разного типа достаточно для старта.

Скиллы в Claude Code: стабильный результат

Скилл — встроенная инструкция которая применяется в каждой сессии. Фиксирует стиль навсегда.

Без скилла

Каждый раз объясняете стиль. Результат непредсказуем. Много итераций.

Со скиллом

Один запрос — ожидаемый результат. Стиль не уплывает.

Как установить: Customize → Create New Skill → Upload ZIP. Не более 2–3 скиллов одновременно — могут конфликтовать. Можно объединить два скилла в один.

Frontend Design skill

Скилл для профессионального дизайна — 3D-анимации, типографика, интерактивные секции.

→Подключение: «...для дизайна используй skill Frontend Design»
→Модель: Sonnet — справляется с дизайном и кодом, бережёт лимиты
→Промпт: «Создай сайт для [ваш бизнес] — название, слоган, логотип, полный лендинг»
→~20 минут — сайт с анимацией, градиентами, профессиональной типографикой
Opus — для финальной полировки, не для старта. Sonnet делает отличный дизайн и экономит лимиты для правок.

Двухэтапный подход

Нельзя смешивать дизайн и маркетинг в одном запросе — Claude будет компромиссить.

1Этап дизайна: «Создай красивый лендинг с Frontend Design skill. Тексты — заглушки.»
2Этап маркетинга: отдельный запрос → «Оцени тексты как профессиональный маркетолог. Что отталкивает клиентов?»
3Переработка: «Теперь перепиши все тексты по найденным проблемам»
Двухэтапный подход даёт и красивый дизайн, и продающие тексты. Одноэтапный жертвует одним ради другого.

Типичные проблемы текстов на сайтах

  • Слоган описывает компанию — нужно описывать результат для клиента
  • Услуги перечислены технически — клиент думает о своей боли, не о вашем процессе
  • Статистика без контекста — «10 лет» ничего не значит без «благодаря чему»
  • CTA «Контакты» — вместо «Обсудить задачу» или «Получить расчёт»
  • Нет конкретики — «высококачественные услуги» vs «сайт за 3 дня с гарантией»
Claude находит эти проблемы и переписывает тексты в один дополнительный запрос.

Три модели работы с клиентом

Разовая сдача

Сделал — передал — получил оплату. Просто, но нет дохода после.

Правки за доплату

База + правки по часовой ставке. Предсказуемо для обеих сторон.

Сопровождение

Ежемесячная оплата. Вы поддерживаете и развиваете. Стабильный доход.

Три способа передачи проекта: папка с файлами / деплой на сервер клиента / полное сопровождение. Оформить договор и самозанятость — Claude помогает создать шаблон, проверьте самостоятельно.

Почему AI-код уязвим

Claude оптимизирован на решение задачи, а не на защиту от злоумышленников.

Аналогия: архитектор спроектировал красивый дом без замков на дверях. Дом работает, но небезопасен.

Статистика исследований:

  • 45–62% AI-кода содержит реальные уязвимости
  • 20% кода пробует установить несуществующие пакеты
«Читайте AI-код как код незнакомого стажёра — с любопытством, но не слепым доверием»

3 типовых сценария утечки

А
API-ключи в клиентском JS

Видны через DevTools любому пользователю → за 48 часов баланс сгорел, карта заблокирована

Б
База данных без Row Level Security

Данные доступны без авторизации → утекло более миллиона токенов данных пользователей

В
Загрузка файлов без проверки

Публичный бакет без ограничений → тысячи приватных документов стали общедоступными

Общий вывод: «AI сделал именно то, о чём его попросили. Никто не спросил — а безопасно ли это?»

Секреты в коде: как правильно

❌ Опасно:

API_KEY = "sk-ant-..."
client = Anthropic(api_key=API_KEY)

✓ Правильно:

import os
client = Anthropic(
  api_key=os.getenv("ANTHROPIC_KEY")
)
Таймлайн после git push с ключом: 0 мин — залили. 2 мин — боты-сканеры нашли ключ. 10 мин — кто-то его использует. ~1 час — счёт на $3000 за чужие запросы.
Правило: ключи в чат не вставлять. Писать YOUR_KEY_HERE, вставить вручную потом.

SQL-инъекции и XSS

❌ SQL-инъекция:

query = f"SELECT * FROM users
WHERE name = '{user_input}'"

Пользователь пишет не имя, а SQL-команду — база её выполняет

❌ XSS:

element.innerHTML = userInput

= разрешить незнакомцу написать что угодно на вашей странице

Решение: параметризованные SQL-запросы вместо f-string. textContent вместо innerHTML. Попросите Claude использовать безопасные методы явно.

Slopsquatting: галлюцинации пакетов

Claude иногда предлагает несуществующий пакет. Хакер его регистрирует — внутри вредоносный код.

Примеры: @types-node вместо @types/node — тире вместо слеша. Визуально похоже, функционально — чужой код.
1Перед npm install — проверьте на npmjs.com или pypi.org
2Смотрите: возраст пакета, количество скачиваний, история автора
3После установки запустите npm audit

Метод двойного прохода + CLAUDE.md

1«Напиши функцию регистрации» → получаем рабочий код
2«Ты — эксперт по кибербезопасности. Проверь по OWASP Top 10 и исправь» → Claude находит и исправляет дыры
Одно дополнительное сообщение кардинально меняет качество кода.
CLAUDE.md — файл в корне проекта. Claude читает его автоматически в каждой сессии. Напишите туда правила: секреты в .env, параметризованные запросы, bcrypt для паролей — и Claude будет следовать им всегда.

Ключевые выводы

  • AI-код рабочий, но не безопасный по умолчанию
  • Секреты — только в .env, никогда в коде или чате
  • Slopsquatting: проверяйте пакеты до npm install
  • Двойной проход + CLAUDE.md = стабильная безопасность

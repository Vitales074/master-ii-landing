Три аудитории —
три уровня доверия

Близкие

Доверяют без доказательств. Портфолио не нужно — но и платят по-дружески

Знакомые

Нужны примеры. 1–2 скриншота и описание — достаточно для решения

Незнакомые

Нужны кейсы + отзывы + сайт. Без этого решение всегда откладывается

Первые кейсы: проекты для себя + для близких. Попросите написать отзыв — живой убеждает лучше формального

Разным людям нужны разные доказательства. Близкие — доверяют без них. Знакомые — нужны примеры работ. Незнакомые — нужны кейсы, отзывы и профессиональный сайт. Как набить портфолио с нуля: делайте проекты для себя, семьи, друзей. Реальные скриншоты и ссылки. Просите написать отзыв. Три-пять кейсов разного типа — достаточно для первых платных клиентов.

01
Скиллы

Стабильный результат
из одного запроса

Скилл — встроенная инструкция в Code, работает в каждой сессии. Без скилла каждый раз объясняете стиль заново, результат непредсказуем. Со скиллом: один запрос — ожидаемый результат, стиль не уплывает. Установка: Customize, Create New Skill, загружаете ZIP. Не более двух-трёх скиллов — могут конфликтовать, можно объединить в один.

Без скиллаСо скиллом

  • Добавляет: 3D-анимации, интерактивность, анимированный фон, адаптивность
  • Как подключить: «…для дизайна используй skill Frontend Design»
  • Не более 2–3 скиллов одновременно — могут конфликтовать
Скилл для профессионального дизайна: 3D-анимации, продуманная типографика, интерактивные секции. Подключение: в запросе пишете «для дизайна используй skill Frontend Design». Модель Sonnet — она справляется с дизайном и кодом и бережёт лимиты. Opus для финальной полировки, не для старта. Один промпт с названием и нишей — за двадцать минут полноценный брендированный сайт.

02
Двухэтапный подход

Сначала дизайн —
потом маркетинг

Нельзя смешивать дизайн и маркетинг. Claude будет компромиссить между красотой и продающими текстами. Три шага. Первый: задача на дизайн, тексты — заглушки. Второй: отдельный запрос на маркетинговый аудит — Claude оценивает как профессиональный маркетолог. Третий: «перепиши все тексты по найденным проблемам» — один запрос, полная переработка.

Маркетинговый аудит:
Claude как копирайтер

СекцияТипичная проблема
СлоганОписывает компанию — не результат для клиента
УслугиТехнические характеристики, а не выгода клиента
Статистика«50 проектов, 3 года» — нет контекста, не убеждает
CTA«Контакты» вместо «Обсудить проект»

Запрос: «Оцени тексты как маркетолог» → «Теперь переработай по найденным проблемам» — два шага

Пять типичных ошибок которые Claude находит. Слоган описывает компанию вместо результата для клиента. Услуги перечислены технически — клиенту не важен процесс, важна его боль. Статистика без контекста — десять лет без объяснения что это значит. CTA «Контакты» вместо «Обсудить задачу». Нет конкретики — «высококачественный» против «за три дня с гарантией».

с клиентом

МодельДоходПодходит для
Разовая сдачаЕдиновременныйПервых проектов, портфолио
Доплата за правкиНестабильныйНебольших команд
Ежемесячное сопровождениеРегулярныйОсновного бизнеса

Оптимальный старт: разовые проекты для кейсов → постепенный переход к ежемесячным клиентам

Три варианта. Разовая сдача: сделал, передал, оплата. Просто, нет дохода после. Правки за доплату: фиксированная база плюс правки по ставке. Предсказуемо для обеих сторон. Сопровождение: ежемесячная оплата, поддерживаете и развиваете. Стабильный доход. Три варианта передачи проекта: папка с файлами, деплой на сервер клиента, полное сопровождение. Договор и самозанятость — Claude помогает создать шаблон, проверьте самостоятельно.

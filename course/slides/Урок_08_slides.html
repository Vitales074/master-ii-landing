Auto Mode: выполняет сразу. Хорошо для простых изменений — поправить кнопку, изменить цвет. Plan Mode: задаёт вопросы, составляет план, ждёт одобрения, потом выполняет. После написания кода Claude не останавливается. Он запускает приложение в браузере, нажимает кнопки, заполняет формы, смотрит консоль. Находит ошибку — исправляет — снова проверяет. Вы видите это в реальном времени в правой части экрана. Это не магия — он использует те же инструменты разработчика, что и человек.

Запомните главное

Ключевые выводы

  • Code → папка → Trust → модель → режим → задача. Порядок не менять
  • Одна строка описания → Claude пишет, проверяет и исправляет сам
  • Sonnet + High — оптимальная связка. Opus + Smarter — для серьёзных проектов
Первый: Localhost — работает только на вашем компьютере. Это не ссылка для клиента — важный момент. Второй: HTML-файл — статичное приложение, открывается в браузере без сервера. Третий: онлайн-сервер — Netlify или Vercel, публичная ссылка. Четвёртый: Telegram-бот — в мессенджере, подробнее в уроке 11. Пятый: exe-установщик — через Electron или PyInstaller для Windows и Mac.

Sonnet — стандарт для большинства задач. Сайты, приложения, тексты — Sonnet справляется. Opus — для сложной архитектуры, нестандартных задач и финальной полировки. Тратит больше токенов. Effort: Low-Medium для обычного, High для крупного проекта, Smarter — самая медленная, самая умная. Не выбирайте максимум везде — это расходует лимиты быстро.
